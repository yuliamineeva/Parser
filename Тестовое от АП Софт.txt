Реализовать REST-API для целей разбора файла на составные части. API должен принимать на вход файл
 (пусть будет ограничение в 1 Мб) - и либо сразу выдавать результат, либо выдавать id, по которому можно
 будет загрузить результат. Файл является текстовым с возможными переводами строк в стиле CRLF, CR и/или LF. 
В начале каждой строки может быть признак начала раздела или подраздела - символ '#'.
 Количество символов '#' в начале строки указывает на уровень вложенности раздела. Результатом должен идти 
разобранный файл в виде набора строк + структура разделов, посредством которой можно будет осуществлять навигацию 
по разделам (позиционирование на строки) на стороне клиента.
